Авторизация и вход в систему - ручное тестирование

Описание проекта:
В рамках проекта выполнено ручное тестирование функциональности входа и авторизации пользователя в веб-приложении. 
Цель проекта - отработка навыков составления QA - артефактов и тестирования позитивных, негативных и граничных сценариев.

Область тестирования: 
Форма входа в систему
Проверка авторизации пользователя
Обработка ошибок при вводе некорректных данных
Валидация данных 

Условия:
Пользователь может войти в систему с валидным email и паролем
Поля Email и Пароль являются обязательными
Пароль должен содержать не менее 8 символов
После 3 неуспешных попыток входа пользователь блокируется на сутки
При вводе некорректных данных отображается сообщение об ошибке

Используемые техники тест-дизайна:
Эквивалентное разбиение
Анализ граничных значений
Негативное тестирование

QA-артефакты:
Чек-лист
Тест-кейсы
Баг-репорты

Структура данных:
Таблица session
Поля:
user_email
first_name
last_name
